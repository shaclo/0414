## [v1.1.6] - 2026-04-29
### 升级主题：节奏闭环 · 颗粒度事件化 · 极致人设 · 阶段隔离 · 血肉主 prompt 直改
- **解节奏与颗粒度问题**：引入事件化 CPG 颗粒度，同一场景停留上限硬约束（连续 ≤2 集），以及骨架阶段的 ITE 闭环自动裁剪冗余节点，显著提高信息密度。
- **强化转折和钩子**：引入钩子类型分类与配比（信息钩、反转钩、动机钩、危机钩），避免全是“危机定格”单一套路，同时设立主线节奏锁强制大反派早日登场。
- **解平淡增强人设**：扩充了 `character` 对象的 `signature_traits` 和 `arc_outline` 字段，并在血肉阶段（BVSR）强制输出 `character_micro_change` 微变化点。
- **CP 互动引擎与规则隔离**：引入并规范 293 条 `cp_interaction_templates`。建立严格的 **阶段隔离铁律**，即“骨架阶段不写感情戏（禁 CP 关键字），CP 互动在血肉阶段按配额和场景抽样注入，扩写阶段只对 Beat 已有内容作镜头化”。
- **缩减灌水描写**：扩写阶段强制限制环境描写字数占比 ≤15%，纯氛围短于 3 行，强化短剧强情节特质。

## 一、骨架阶段（Phase 2 — Skeleton）

### 1.1 全新节点详情 & AI 交互编辑对话框（`node_detail_dialog.py` — 新增 1322 行）

- **手动编辑**：双击节点打开详情对话框，直接编辑标题、环境、角色、事件摘要、情感基调、结尾钩子等全部 6 个字段。
- **Chat 对话模式**：与 AI 就某一集内容进行自由讨论，AI 区分"讨论"和"修改建议"两种回复模式，只有用户明确确认后才执行修改。
- **Quick-Regen 快速重生成**：提供"戏剧方向"（更激烈/更复杂/更温情/更悬疑/更黑暗/更紧凑）和"结构调整"（增加反转/强化钩子/增加内心冲突/聚焦主线）等预设选项，一键重新生成节点内容。
- **BVSR 重写**：以指定人格视角完全重构本集内容，保持 Hauge 阶段和因果关系约束。
- **版本管理**：每次修改自动创建新版本快照（含时间戳和来源标记），版本下拉可切换回溯。

### 1.2 节点拆分功能（`split_dialog.py` — 新增 296 行）

- 支持将一集拆分为 2~5 个片段。
- 每个事件通过下拉框可视化分配到目标片段。
- 拆分后处理模式可选：仅拆分 / AI 补全（补标题/钩子/情感基调）/ AI 扩写。
- 拆分后自动更新 CPG 因果边（原节点的入边接到第一个片段，出边接到最后一个片段，片段间自动串联）。

### 1.3 级联改写后续章节（`cascade_rewrite_dialog.py` — 新增 542 行）

- 修改一个章节后，自动调整后续 N 章（最多 5 章）以保持因果连贯。
- **两种模式**：
  - **仅调整开头衔接**：只修改后续章节的开头 1-2 个事件使其与上文衔接，保持结尾钩子和主要冲突不变。
  - **完整级联改写**：全面调整后续章节的事件、角色、情感和钩子。
- 改写结果以卡片列表展示，支持逐章预览和应用。

### 1.4 骨架编辑器（CPG Graph Editor）大幅升级

- 卡片布局重构：增加节点卡片的视觉信息密度，显示环境、角色、事件摘要、情感基调、结尾钩子等详细内容。
- 支持右键菜单：编辑 / 删除 / 拆分 / 级联改写 / 版本管理等操作。
- 加入 Loading 动画覆盖层，骨架生成期间显示加载状态。
- 新增集数 (`total_episodes`) 和每集时长 (`episode_duration`) 设置控件。
- 节点编号冲突检测：进入下一阶段前自动检查重复编号。
- 每个新生成的节点自动创建 v0 版本快照。
- 新增 `episode_hook`（结尾钩子）字段。

### 1.5 骨架 AI 设置对话框（`skeleton_ai_settings_dialog.py` — 新增 359 行）

- 独立的骨架阶段 AI 参数调整面板（温度、Top-P、Top-K、Max Tokens）。
- 通过菜单栏「设置 → 骨架 AI 设置」访问。

## 三、血肉阶段（Phase 3 — Flesh）

### 3.1 批量 & 全自动模式

- **批量模式**：连续生成多个章节的 Beat，每确认一个后自动开始下一个。
- **全自动模式**（✈️）：自动选择第一个有效 Beat 变体并确认，无需人工干预，全部完成后自动进入扩写阶段。
- 新增批量数量 SpinBox 控件。

### 3.2 Beat 可读视图

- 新增"可读视图"和"JSON 编辑"双视图切换按钮。
- 可读视图将 Beat JSON 渲染为结构化的剧本摘要格式：场景描述 / 活跃角色 / 因果事件链（带序号图标和因果指示） / 悬念钩子 / 创作理由。

### 3.3 因果关系上下文注入

- Beat 生成时注入 `edge_relations_context`（因果连线关系），让 AI 理解节点间的前后因果。
- 结合 `drama_style_block`（风格块）注入，确保生成内容符合剧集风格设定。

## 四、扩写阶段（Phase 5 — Expansion）

### 4.1 因果图驱动的前情上下文

- 扩写不再按列表顺序取"上一节点"作为前情，而是通过因果图的入边 (`incoming_edges`) 获取真正的前情来源。
- Prompt 中注入完整的因果连线关系描述和前序节点的结尾片段（最后 500 字），大幅提升场景衔接质量。
- 新增 `episode_number`（集编号）字段注入，用于正确标记集标题头。

### 4.2 批量扩写改为全量重写

- "批量扩写"功能改为"批量全部重写"——重置所有已扩写状态后全部重新生成。
- 新增覆盖警告提示，告知用户有多少节点已有文本将被覆盖。

### 4.3 导出格式优化

- 导出时按节点编号数字排序。
- 已扩写的节点直接输出文本，未扩写的显示占位符。
- 新增"保存当前节点"和"重新生成后续节点"按钮。

### 4.4 爽感 & 钩子公式系统（v1.1.0 新增）

- **爽感公式管理**：新增 `prompt_templates.json` 存储爽感公式（小爽/中爽/大爽），通过 `prompt_template_dialog.py` 管理对话框启用/禁用/编辑。
- **钩子公式管理**：新增 7 种结尾钩子公式（动作截断/身份炸弹/危机降临/异变暗示/选择悬念/暗处窥视/隐喻细节），支持多选注入。
- **爽感节奏调度**：新增 `determine_satisfaction_level()` 方法，按集号自动调度小爽/中爽/大爽等级（可配置间隔参数 SpinBox）。
- **时长区间控制**：新增 `range_slider.py`（双端滑块，294 行），支持设定每集时长范围并自动换算目标字数。
- **爽感等级实时提示**：选择节点时自动显示本集应使用的爽感等级。

### 4.5 爽感 & 钩子写作铁律（v1.1.0 新增，永久注入 System Prompt）

- **爽感写作铁律**：强制 AI 使用「铺垫压迫 → 反差释放 → 旁观者放大」三段式技法，禁止"无铺垫的胜利"等伪爽感模式。
- **钩子写作铁律**：强制每集最后 3-5 行为"正在发生的动作被中断"，禁止"角色陷入沉思""旁白式预告"等伪钩子模式。

## 更新总结

v1.1.2 相比 Beta v1 的核心升级集中在以下方面：

1. **骨架阶段深度交互**：从简单的生成+查看升级为可拆分、可级联改写、可 Chat 对话、可多版本管理的全方位骨架编辑系统。
2. **人物关系可视化**：新增力导向关系图，角色管理从纯列表升级为图文并茂的双视图模式。
3. **血肉阶段自动化**：支持批量和全自动模式，新增 Beat 可读视图，大幅降低人工操作成本。
4. **扩写因果图驱动**：扩写上下文从线性顺序改为因果图驱动，场景衔接质量显著提升。
5. **爽感 & 钩子系统**：公式化管理 + 铁律级 System Prompt 约束 + 按集号自动调度，确保每集有节奏感的高潮和悬念。
6. **系统自动更新**：支持从 GitHub Release 一键检查/下载/更新，版本号自动写入本地。
7. **系统可配置性**：新增 BVSR 人格管理、题材预设、骨架 AI 设置、爽感/钩子公式设置等配置面板，全部支持持久化。
8. **版本管理**：节点级版本快照系统，支持修改回溯。

## 九、用户反馈意见与落地情况

以下是用户对 v1.0 版本使用后的核心反馈，以及对应的实现方案和完成状态。

### 反馈 1：单集事件数量过多，短剧观众易疲劳

> **原文**：血肉中事件数量有多有少，最多见到过单集 5 件事（悬疑类）。短剧最好每集只解决一件事。也可以考虑从项目开始就分化为长视频逻辑和短剧逻辑。

| 项目 | 内容 |
|------|------|
| **完成状态** | ✅ 已实现 |
| **实现方式** | 通过「节点拆分功能」（`split_dialog.py`）解决——当一集事件过多时，可将其拆分为 2~5 个片段，每个片段聚焦一件事。拆分后自动更新因果边，保持剧情连贯。 |
| **对应文件** | `ui/widgets/split_dialog.py`（296 行） |
| **长短剧分化** | 暂未实现独立的长/短剧逻辑模式切换。当前通过题材预设（`genre_presets.json`）部分实现：短剧预设场景数 1-2、长剧预设 2-4。未来可考虑在项目创建时增加「短剧模式 / 长剧模式」开关。 |

### 反馈 2：单集时长不要严格限制，应支持灵活区间

> **原文**：单集事件不设太严格的上限，1.5-5 分钟都可以接受。重要事件用长篇幅，过场事件短篇幅快过，不用严格卡 3 分钟。

| 项目 | 内容 |
|------|------|
| **完成状态** | ✅ 已实现 |
| **实现方式** | 新增「双端滑块」（`range_slider.py`），用户可设定每集时长的**区间**（如 1.5~5 分钟），而非固定值。系统根据区间中点自动换算目标字数（每分钟约 180 字），并在 Prompt 中注入目标字数范围。 |
| **对应文件** | `ui/widgets/range_slider.py`（294 行）、`ui/phase5_expansion.py` 中的 `_duration_slider` 及 `_update_word_count_hint()` |
| **效果** | AI 扩写时收到的是字数区间而非固定值，允许根据事件重要性弹性调整篇幅。 |
